Week 7 Capstone

This round marked a full rewrite of the optimisation pipeline into a single, modular, per-function framework, replacing the round-by-round patching of a single script. The pipeline now dispatches each function to one of several modes (standard GP-BO, multi-kernel, TuRBO, or coverage sampling) based on a per-function configuration, with the transform, kernel, acquisition function, and SVM filter all configurable per function. The substantive changes this round were:
i) Per-function output transforms refined. Transforms are now selected per function to match each one's output distribution:
* Functions 2, 3, 6, 8: standardisation (subtract mean, divide by standard deviation).
* Function 5: changed to log scaling, reflecting its output spanning several orders of magnitude (0.1 to ~1834); log scaling gives a well-distributed target the GP can fit, rather than letting the largest values dominate.
* Functions 4 and 7: changed to Yeo–Johnson power transform, to compress their heavy-tailed output distributions toward normality.
* Function 1: signed-log transform (carried forward), though function 1 is now handled by coverage sampling rather than BO (see iv).
ii) Multiple kernels for higher-dimensional functions. Rather than a single RBF, functions now use ARD Matérn kernels (separate lengthscale per dimension) with a ConstantKernel signal-variance term and a learnable WhiteKernel for noise. For functions 6 and 7, a multi-kernel Thompson-sampling approach was introduced: separate GPs are fitted with Matérn-1.5, Matérn-2.5, and RBF bodies, one posterior sample is drawn per candidate per kernel, and the candidate with the highest sample across all kernels is selected. This hedges against committing to a single (possibly wrong) assumption about the function's smoothness, which matters more in higher dimensions where the surrogate is less certain. This round, function 6 selected the RBF kernel and function 7 selected Matérn-2.5.
iii) TuRBO for function 8. Function 8 (8-dimensional) now uses a trust-region Bayesian optimisation approach. A trust region is maintained around the current best point and adapts in size based on the success/failure history of recent evaluations; the local GP and acquisition operate only inside this box. This is better suited to the high-dimensional setting than a global GP, which struggles to model an 8-D space with the available points. This round the trust-region length was 0.0625 (already shrunk through several failure steps), and the proposal sits 0.058 from the current best — local exploitation.
iv) Function 1 moved to coverage sampling. After repeated rounds where the surrogate could not fit function 1's sparse-signal data (kernels collapsing to a noise-dominated fit), function 1 was removed from the BO process. It now uses Sobol maximin coverage — selecting the candidate furthest from all existing observations — to systematically fill the unsampled regions of the input space, pending the discovery of any region with genuine signal.
v) SVM search-space filter added. For functions 3–8, a support vector machine classifier is trained each round to label the input space as "likely high-Y" or "likely low-Y" (using the top 25–30% of observed values as the positive class), and candidate points classified as low-Y are excluded before the acquisition maximum is taken. This narrows the search to the region the data suggests is promising — particularly valuable for the higher-dimensional functions where the full space is too large to search effectively. This round the filter retained between 8% and 25% of the candidate grid depending on the function.
vi) Acquisition evaluated on a Sobol/Latin Hypercube candidate set. The acquisition function (UCB, ? = 1.96) is evaluated over 10000 quasi-randomly sampled candidate points, with the exclusion radius (from Round 6) and the SVM filter both applied before the maximum is selected
